Amount of words in commit message

What is it?

This is visualization of amount of word occurrences in commit messages over history of project. Inspired by this TED talk.
Visualization data is based on these csv files generated with code history mining CLI.

The idea is to take all words from commit message, filter out words unrelated to change (e.g. prefix for issue tracker or author name) and count how often words mentioned within a week/month. Absolute value of word occurrences is interesting on its own to see what happened on project. Obviously, if some word is used in commits often, it will strongly correlate with any activity on the project. Considering this it might be worth looking for relative difference between words and how it changes over time.
